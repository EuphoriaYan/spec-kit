# v0.12.5+teamwork.2

[English backup](v0.12.5-teamwork.2_en.md)

`v0.12.5+teamwork.2` 是基于上游 Spec Kit `v0.12.5` 的第二个固定 AI Team
发行版，也是首个完整包含当前六个主 Team Skills 和高级 Memory 扩展入口的固定版本。

## 发行范围

- 为 Codex、Claude Code、Cursor 和 Trae 提供默认的 Team 最小安装模式；
- 安装 `Specify`、`Plan-and-Task`、`Assess`、`Fix`、`Implement` 和 `Review`
  六个职责明确的主 Team Skills；
- 提供 `Memory Consolidate` 高级扩展入口，用于显式的交付后经验沉淀和知识晋升，
  不参与 Feature/Bugfix 自动路由；
- Feature 使用 Specify -> Plan-and-Task -> Implement 路径，Bugfix 使用
  Assess -> Fix 路径，二者统一进入 Review/Assess/Fix 质量循环；
- Plan 和缺陷分析直接使用 CodeGraph 判断源码影响范围；
- Feature 与 Bugfix 过程包默认保留在本地，不随业务代码提交；
- 面向人的中文主文档与 `_en` 英文备用文档同步发布。

## 固定信息

- Git tag：`v0.12.5+teamwork.2`
- Python package version：`0.12.5+teamwork.2`
- 上游基线：`github/spec-kit@v0.12.5`
- CodeGraph CLI：`>=1.0.0,<2.0.0`
- Python：`>=3.11`

## 安装

```bash
uv tool install specify-cli --force \
  --from git+https://github.com/EuphoriaYan/spec-kit.git@v0.12.5+teamwork.2
```

随后在真实代码仓根目录执行：

```bash
specify init . --integration <codex|claude|cursor-agent|trae>
```

从 `v0.12.5+teamwork.1` 或临时 `main` 安装升级时，重新执行上述安装和初始化，
然后检查六个主 Skills、高级扩展入口、受管理的 AI 规则入口、`.gitignore` 和项目自有
Team 配置。
如需回滚，重新安装上一个已审核 tag，并在覆盖项目前比较初始化产生的差异。
